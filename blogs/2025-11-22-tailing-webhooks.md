<p>If you run an application that produces a lot of webhooks like <a href="https://emailengine.app/?ref=blog.emailengine.app">EmailEngine</a>, you might want to observe what kind of data is even sent before you try to start consuming it. This post shows an easy way to do this with PHP.</p><p>The concept is simple. Your webhooks producer posts all webhooks to a PHP script. That script then appends all incoming JSON requests with some metadata to a log file. You would tail that log file and pipe it to the <em>jq</em> command, resulting in a real-time pretty printed log output. As the content is stored in a log file, you can stop tailing any time you want and return to it later.</p><p>First, if you do not have <em>jq</em> yet installed, you can add it using your package manager. For example, in Ubuntu, you can run the following:</p><pre><code>$ sudo apt update
$ sudo apt install -y jq
</code></pre><p>Next, create a log file. I&apos;ll be running PHP scripts under the <code>www-data</code> user, so I need to make sure PHP can write to that file.</p><pre><code class="language-bash">$ sudo touch /var/log/whlog
$ sudo chown www-data /var/log/whlog</code></pre><blockquote>Make sure that this file is empty. Otherwise, jq will fail to process it if it contains non-JSON data.</blockquote><p>Next, get the example PHP script from <a href="https://gist.github.com/andris9/e1ad312ef25c46dd9397d2726995581a?ref=blog.emailengine.app">here</a> and, if needed, change the log file path at the beginning of the script. Store this script somewhere in your web server so that you would be able to run requests against it.</p><p>Now, add the PHP script as the webhook destination in your app, in this case, for EmailEngine.</p><figure class="kg-card kg-image-card"><img alt="alt" class="kg-image" height="560" src="https://blog.emailengine.app/content/images/2022/06/Screenshot-2022-06-22-at-09.14.05.png" width="1382" /></figure><p>In the server where the log file resides, run the following command to tail the log output.</p><pre><code>$ tail -f /var/log/whlog | jq
</code></pre><p>And finally, to test if everything works, send a test webhook payload from EmailEngine.</p><figure class="kg-card kg-image-card"><img alt="alt" class="kg-image" height="580" src="https://blog.emailengine.app/content/images/2022/06/Screenshot-2022-06-22-at-09.13.39.png" width="1396" /></figure><p>If you get a &quot;Test payload sent&quot; response, then EmailEngine managed to send out the webhook.</p><figure class="kg-card kg-image-card"><img alt="alt" class="kg-image" height="611" src="https://blog.emailengine.app/content/images/2022/06/Screenshot-2022-06-22-at-09.14.23.png" width="1288" /></figure><p>Check the terminal window where you are tailing the log file. You should see the example webhook payload. Anything the server sent is inside the &quot;request&quot; property.</p><figure class="kg-card kg-image-card"><img alt="alt" class="kg-image" height="868" src="https://blog.emailengine.app/content/images/2022/06/Screenshot-2022-06-22-at-09.16.26.png" width="1644" /></figure><p>That&apos;s it! You can now tail any kind of webhooks in real-time, as long as these are JSON formatted.</p>
